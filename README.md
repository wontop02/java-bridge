# 🖥️ 우테코 최종 미션 연습 - [다리 건너기]

- 기능 요구 사항에 기재되지 않은, 스스로 판단한 부분은 **볼드체** 사용

## 📖 프로젝트 개요

- 위아래 둘 중 하나의 칸만 건널 수 있는 다리를 끝까지 건너가는 게임
- 위아래 두 칸으로 이루어진 다리를 건너야 함
    - 다리는 왼쪽에서 오른쪽으로 건넘
    - 위아래 둘 중 하나의 칸만 건널 수 있음
- 다리의 길이를 숫자로 입력받고 생성
    - 다리를 생성할 때 위 칸과 아래 칸 중 건널 수 있는 칸은 0과 1 중 무작위 값을 이용해 정함
    - 위 칸을 건널 수 있는 경우 U, 아래 칸을 건널 수 있는 경우 D값으로 나타냄
    - 무작위 값이 0인 경우 아래 칸, 1인 경우 위 칸이 건널 수 있는 칸
- 다리가 생성되면 플레이어가 이동할 칸 선택
    - 이동할 때 위 칸은 대문자 U, 아래 칸은 대문자 D 입력
    - 이동한 칸을 건널 수 있으면 O로 표시, 건널 수 없으면 X로 표시
- 다리를 끝까지 건너면 게임 종료
- 다리를 건너다 실패하면 게임을 재시작하거나 종료
    - 재시작할 경우 처음 만든 다리 재사용
    - 게임 결과의 총 시도한 횟수는 첫 시도를 포함해 게임을 종료할 때까지 시도한 횟수를 나타냄
- 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`을 발생시키고,
  "[ERROR]"로 시작하는 에러 메시지 출력 후 그 부분부터 다시 입력 받음

## ⌨️ 입출력 요구 사항

### 입력

- 자동으로 생성할 다리 길이 입력
    - 빈 문자열이 입력된 경우 예외 발생
    - 숫자를 제외한 문자가 입력된 경우 예외 발생
    - int 범위를 벗어난 숫자가 입력된 경우 예외 발생
    - 3 이상, 20 이하의 숫자가 아닐 경우 예외 발생
- 플레이어가 이동할 칸 입력
    - U(위 칸)와 D(아래 칸) 중 하나의 문자 입력
    - 빈 문자열이 입력된 경우 예외 발생
    - U와 D를 제외한 문자가 입력된 경우 예외 발생
- 게임 재시작/종료 여부 입력
    - R(재시작)과 Q(종료) 중 하나의 문자 입력
    - 빈 문자열이 입력된 경우 예외 발생
    - R과 Q를 제외한 문자가 입력된 경우 예외 발생

### 출력

- 게임 시작 문구
- 최종 게임 결과
    - 사용자가 이동할 수 있는 칸을 선택한 경우 O 표시
    - 사용자가 이동할 수 없는 칸을 선택한 경우 X 표시
    - 선택하지 않은 칸은 공백 한 칸으로 표시
    - 다리의 시작은 `[`, 끝은 `]`로 표시
    - 다리 칸의 구분은 `|`(앞뒤 공백 포함) 문자열로 구분
- 게임 종료 문구
- 에러 메시지
    - "[ERROR]"를 붙여서 출력

## 📜 프로그래밍 요구사항

### InputView 클래스

- 제공된 `InputView` 클래스를 활용해 구현
- 패키지 변경 가능
- 메서드의 시그니처(인자, 이름)와 반환 타입 변경 가능
- 필요한 메서드 추가 가능

### OutputView 클래스

- 제공된 `OutputView` 클래스를 활용해 구현
- 패키지 변경 가능
- 메서드명 변경 불가, 인자와 반환 타입은 추가, 변경 가능
- 필요한 메서드 추가 가능

### BridgeGame 클래스

- 제공된 `BridgeGame` 클래스를 활용해 구현
- 필드(인스턴스 변수) 추가 가능
- 패키지 변경 가능
- 메서드명 변경 불가, 인자와 반환 타입은 추가, 변경 가능
- 필요한 메서드 추가 가능

### BridgeMaker 클래스

- 제공된 `BridgeMaker` 클래스를 활용해 구현
- 필드(인스턴스 변수) 변경 불가
- 메서드명, 시그니처(인자, 이름)와 반환 타입 변경 불가

### BridgeRandomNumberGenerator 클래스

- Random 값 추출은 제공된 `bridge.BridgeRandomNumberGenerator`의 `generate()`를 활용
- 사용 예시
    - 다리 칸을 생성하기 위한 Random 값은 아래와 같이 추출

```
int number = bridgeNumberGenerator.generate();
```

### 라이브러리

- `camp.nextstep.edu.missionutils`에서 제공하는 `Console` API를 사용하여 구현
    - 사용자가 입력하는 값은 `camp.nextstep.edu.missionutils.Console`의 `readLine()`을 활용

## ✅ 구현 기능 목록

### 시작 문구 출력

```
다리 건너기 게임을 시작합니다.
```

### 다리 길이 입력

```
다리의 길이를 입력해주세요.
3
```

- 사용자에게 입력값 요청
    - `camp.nextstep.edu.missionutils.Console`의 `readLine()` 활용
- 입력값 검증: 잘못된 값일 경우 `IllegalArgumentException` 발생 및 재입력 요청
    - 빈 문자열이 입력된 경우 예외 발생
    - 숫자를 제외한 문자가 입력된 경우 예외 발생
    - int 범위를 벗어난 숫자가 입력된 경우 예외 발생
    - 3 이상, 20 이하의 숫자가 아닐 경우 예외 발생

### 다리 생성

- 위 칸과 아래 칸 중 건널 수 있는 칸 정함
    - `bridge.BridgeRandomNumberGenerator`의 `generate()`를 사용해 랜덤 값 추출
    - 0이 나올 경우 아래 칸, 1이 나올 경우 위 칸이 건널 수 있는 칸이 됨

### 이동할 칸 입력

```
이동할 칸을 선택해주세요. (위: U, 아래: D)
U
```

- 사용자에게 입력값 요청
    - `camp.nextstep.edu.missionutils.Console`의 `readLine()` 활용
- 입력값 검증: 잘못된 값일 경우 `IllegalArgumentException` 발생 및 재입력 요청
    - U(위 칸)와 D(아래 칸) 중 하나의 문자 입력
    - 빈 문자열이 입력된 경우 예외 발생
    - U와 D를 제외한 문자가 입력된 경우 예외 발생

### 다리 이동

- 사용자 입력에 따라 다리 이동
    - 사용자 입력과 다리가 같을 경우 성공: O
    - 사용자 입력과 다리가 다를 경우 실패: X

### 다리 출력

```
[ O | X ]
[   |   ]
```

- 이동할 칸을 한 번 선택할 때마다 다리 상태 출력

### 재시작 여부 입력

```
게임을 다시 시도할지 여부를 입력해주세요. (재시도: R, 종료: Q)
R
```

- 다리 건너기에 실패한 경우(X가 나온 경우) 재시작 여부 입력
- 사용자에게 입력값 요청
    - `camp.nextstep.edu.missionutils.Console`의 `readLine()` 활용
- 입력값 검증: 잘못된 값일 경우 `IllegalArgumentException` 발생 및 재입력 요청
    - R(재시작)과 Q(종료) 중 하나의 문자 입력
    - 빈 문자열이 입력된 경우 예외 발생
    - R과 Q를 제외한 문자가 입력된 경우 예외 발생
- 재시작할 경우 시도 횟수 증가

### 최종 게임 결과 출력

```
최종 게임 결과
[ O |   |   ]
[   | O | O ]

게임 성공 여부: 성공
총 시도한 횟수: 2
```

- 실패한 상황에서 종료한 경우: 실패
- 다리 건너기에 성공한 경우: 성공
